# 前言

2024 年下半年，我开始在公司内部推动 AI Agent 的落地。事情进展得并不顺利。

问题不在于"做不出来"。相反，Demo 做得非常快——用 LangChain 搭一个能调用工具、能多轮对话的 Agent，一个下午就够了。真正的困难在后面：怎么让它稳定？怎么评估它的质量？什么时候该让它自主决策，什么时候该让它停下来等人确认？当它"胡思乱想"的时候，到底是提示词的问题、工具的问题，还是模型本身的问题？

这些问题没有现成答案。市面上的资料大多是两种极端：要么是 5 分钟 Quickstart，教你调一个 API 就跑起来，然后告诉你"剩下的自己看文档吧"；要么是学术论文，讨论 ReAct、CoT、ToT 的各种变体，但对一个真实的生产系统要怎么搭、怎么测、怎么修，只字不提。

这本书就是来填这个坑的。

## 这本书讲什么

一句话概括：**从第一性原理出发，讲清楚怎么把一个 AI Agent 从原型做到生产级。**

全书 21 章，分为五篇：

**第一篇 原理篇（第1-4章）** 建立认知框架。Agent 的本质是什么？ReAct 循环为什么是 Agent 的心跳？当前的技术生态有哪些框架和工具？这部分不写代码，但会让你理解 Agent 的"为什么"——之后所有章节都是在这个框架上添砖加瓦。

第二篇 开发篇（第5-9章） 动手实战。从零写一个最小 ReAct Agent，再到 LangChain 和 LangGraph 的深度实战，覆盖工具调用、记忆系统等核心能力。这是全书最重的一篇——你会亲手写出多个可运行的 Agent。

第三篇 增强篇（第10-13章） 让 Agent 从"能用"到"好用"。高级推理策略、RAG 知识增强、多 Agent 协作、人机协同——这些技术决定了你的 Agent 是"勉强及格"还是"超出预期"。

第四篇 生产篇（第14-18章） 解决工程落地问题。评估怎么设计？安全怎么防？部署怎么选？调试怎么定位？最后用行业案例拆解，看看别人是怎么做的。

第五篇 前沿篇（第19-21章） 站远一点看方向。Agent 的商业模式、技术趋势、学习路线图——这不是技术手册，而是帮你建立对 Agent 领域的长期判断力。

每一章都遵循相同的结构：章节引言 → 理论讲解 → 代码实战 → 进阶拓展 → 开放讨论 → 进阶必做。你不一定要按顺序读完——可以根据自己的需求跳读。

## 这本书不适合谁

先说清楚，免得浪费你的时间。

- 如果你想要一本"API 速查手册"，这本书不合适。我们讲的是原理和工程实践，不是接口文档。
- 如果你没有任何编程经验，这本书也不合适。你需要具备 Python 基础（函数、类、异步），了解 HTTP API 的基本概念，能在命令行里跑脚本。
- 如果你想学机器学习、训练自己的模型，这本书同样不合适。我们调用 API，不训练模型。

但你**不需要**懂 LangChain 或 LangGraph——我们从零讲起。也不需要前端经验——TypeScript 部分是可选的补充内容。更不需要机器学习背景——Agent 开发的核心是工程能力，不是算法能力。

## 这本书的写法

你可能已经注意到，市面上很多 AI 技术书有惯性套路：

- 先用大段篇幅歌颂 AI 的伟大意义
- 再堆砌一堆 API 参数表格
- 最后给你一个"Hello World"级的 Demo，套个壳就收工

我们不这么写。

**第一，从第一性原理出发。** 每引入一个概念，先讲"为什么需要它"，再讲"怎么实现它"。你不会在本书中看到 API 参数的罗列——那是在浪费你的时间。你要理解的是机制，不是参数名。

**第二，代码是第一公民。** 所有代码都可以独立运行，我们在文件头标注了框架版本，尽量用 mock 工具而不是逼你去申请 API Key。关键代码片段控制在 30 行以内——我们相信短代码比长代码更有解释力。

**第三，面向真实世界的复杂性。** Agent 开发最难的从来不是写 Demo，而是处理非确定性：同样的问题，Agent 每次回答可能不一样；同样的提示词，换个模型可能效果天差地别。我们不回避这些"不漂亮"的问题——相反，它们才是这本书真正的价值所在。

**第四，融入中国传统智慧。** 这不是为了凑文化调性，而是很多技术原理，古人早已用一句话说透了。比如 ReAct 循环的本质是"思考→行动→观察"交替进行，这和王阳明的"知行合一"如出一辙——不是先知道再行动，而是在行动中深化认知。全书不超过 30 处引用，点到为止。

## 怎么读这本书

如果你是从业工程师，建议按顺序读完前两篇（原理篇 + 开发篇），后面的增强篇和生产篇按需跳读。

如果你正在面试 AI Agent 相关岗位，重点看原理篇和开发篇，关注每章的"开放讨论"——这些问题往往就是面试中的开放题。

如果你已经在做 Agent 产品但遇到了稳定性瓶颈，直接跳到生产篇，评估、安全、部署、调试四章可能是你当下最需要的。

每一章的末尾有"进阶必做"练习题。建议你至少完成前三题的编码——光看不练，和看菜谱学做菜没什么区别。
